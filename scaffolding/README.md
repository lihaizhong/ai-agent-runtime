# Scaffolding

工程搭建的准备工作，负责在项目开始之前提供完整的初始化和配置支持。

## 核心职责

Scaffolding 在项目启动阶段承担以下关键职能：

- **项目初始化**：快速创建项目基础结构，包括目录、配置文件和必要的代码模板
- **模板管理**：提供和维护各类项目模板，支持不同场景的快速启动
- **环境检查**：自动检查开发环境依赖，确保所有必需工具和库已正确安装
- **配置生成**：根据项目需求生成配置文件，包括构建工具、开发工具、测试框架等
- **依赖安装**：自动安装和管理项目依赖，确保版本兼容性和依赖完整性
- **目录结构创建**：按照最佳实践创建标准化的项目目录结构
- **代码规范设置**：初始化代码规范工具，如 ESLint、Prettier、Black 等
- **版本控制初始化**：配置 Git 仓库、.gitignore 文件和提交规范
- **Hooks 配置**：自动配置各类 hooks，包括 git hooks、pre-commit hooks、CI/CD hooks、Coding Agent hooks 等

## 使用场景

Scaffolding 适用于以下场景：

- 创建新的 AI Agent 项目
- 初始化全栈开发项目
- 搭建微服务架构
- 配置多模块 monorepo
- 设置持续集成/持续部署（CI/CD）环境
