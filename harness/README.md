# Harness

工程运行时的治理工作，负责在 AI 模型工作时提供全面的支持和管理。

## 核心职责

Harness 在模型工作过程中承担以下关键职能：

- **工具分发**：智能地将合适的工具分发给模型，根据任务需求动态选择和调用
- **上下文压缩**：管理上下文窗口，智能压缩和优化上下文信息，确保关键内容不被丢失
- **跨伦状态管理**：维护和管理跨对话轮次的状态，保持会话的连续性和一致性
- **安全规则执行**：执行预定义的安全规则和策略，防止模型输出有害或不当内容
- **经验记忆写入**：将对话中的经验和学习内容自动写入记忆系统，供后续会话复用
- **长会话稳定性保障**：监控和优化长会话的运行状态，防止会话崩溃或性能下降

## 工具集成

Harness 配合以下工具进行项目管理与维护：

- **superpowers**：增强模型能力的工具集
- **openspec**：开放规格定义和管理
- **speckit**：规格化工具包

通过这些工具的协同工作，Harness 提供了一个稳定、安全、高效的 AI 运行时环境。
